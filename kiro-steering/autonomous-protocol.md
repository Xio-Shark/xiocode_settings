# AI Agent 自主执行协议

> 适用工具：Claude Code / Codex / Kiro / Copilot / OpenCode 及兼容 Agent 框架

所有涉及项目文件修改且包含 ≥2 个可独立验收步骤的任务，必须遵循本协议。单步查询、纯问答、代码解释、代码审查不触发。

## 执行流程

1. **任务拆解**：拆分为 3–12 条可验收步骤
2. **创建文件**：在项目根目录同时创建 `{任务名} 规划.md` + `{任务名} TO DO list.csv`
3. **自动执行循环**：逐条执行 → 验证 → 从 CSV 删除该行 → 继续下一条
4. **清理**：全部完成后删除 CSV 和规划.md，输出执行摘要

## CSV 格式（统一标准）

```csv
id,group,item,status,notes
1,A,第一步描述,TODO,
2,A,第二步描述,TODO,
3,B,第三步描述,TODO,
```

- `id`：全局递增序号
- `group`：解耦组标识（`A` / `B` / `C`），无分组需求时统一填 `A`
- `item`：具体可验收的执行项（动词开头）
- `status`：`TODO` / `IN_PROGRESS`
- `notes`：备注（文件路径、验证方式等）

**状态机**：仅允许 `TODO` → `IN_PROGRESS`，任意时刻最多 1 行 `IN_PROGRESS`

**完成即删**：每条 item 验证通过后立即从 CSV 删除该行。CSV 始终只反映未完成的工作。

## 分组规则

当项目级 `AGENTS.md` 定义了 "List 文档自动执行协议" 时，遵循其三组解耦拆分规则。无特殊要求时，所有 item 统一归入组 `A`，按顺序执行。

## 异常处理

| 场景 | 处理 |
|---|---|
| 步骤失败 | 记录原因到 notes，暂停报告 |
| 需用户输入 | 追加 `"等待确认: xxx"` 到 notes，暂停 |
| 范围变更 | 只做追加，不重排 |
| 重试 > 5 次 | 改变策略，记录到 notes |

## commit 策略

遵循项目级规定。若项目 `AGENTS.md` 或 `AGENTS.trellis.md` 禁止 AI commit，则 AI 只做代码修改，由人工审核后提交。

## 上下文恢复

中断后恢复：扫描 `* TO DO list.csv` → 按 group 分组 → 找到首个 `TODO` / `IN_PROGRESS` → 读 `* 规划.md` → 恢复执行。

子项目：根目录任务 → 根目录 CSV；子目录任务 → 子目录 CSV。子目录 `AGENTS.md` **优先于本文件**。

输出：`任务: {目标} | 进度: {已完成}/{总数} | 当前: {步骤}` | 完成: `✅ 任务完成: {任务名}`
