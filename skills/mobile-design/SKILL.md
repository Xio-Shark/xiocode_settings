# Mobile Design

## Overview

Design for phones and tablets with mobile-native interaction, performance limits, and platform conventions in mind.

## Rules

- Treat mobile as its own medium, not a smaller desktop.
- Ask for platform and framework when they are unclear.
- Prioritize touch targets, thumb reach, loading/error states, and constrained attention.
- Respect platform-specific conventions when building iOS, Android, or cross-platform UI.

## When to use

- mobile UI design
- touch interaction decisions
- navigation and layout on phones/tablets
- offline-aware or interruption-heavy mobile flows

## Workflow

1. Confirm platform, framework, and device targets.
2. Define primary tasks and their one-handed interaction flow.
3. Place core actions where they are reachable and obvious.
4. Design states:
   - loading
   - empty
   - error
   - offline
5. Validate performance-sensitive UI choices before overcomplicating structure.

## Focus points

- touch target size
- thumb zone placement
- platform navigation expectations
- battery/performance awareness
- concise mobile-first content hierarchy
