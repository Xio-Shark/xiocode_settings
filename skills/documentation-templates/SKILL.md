# Documentation Templates

## Overview

Provide a clean starting structure for common documentation types.

## Rules

- Start with the smallest structure that answers the reader’s first questions.
- Put quick-start material before deep reference material.
- Prefer examples and concrete structure over abstract prose.
- Keep docs updated; stale docs are worse than missing docs.

## When to use

- README creation or cleanup
- API documentation
- ADRs
- changelogs
- developer-facing project docs

## Workflow

1. Identify the document type and audience.
2. Choose the minimal structure for that type.
3. Put the fastest path to understanding near the top.
4. Add examples, references, and maintenance sections only where they help.

## Common outputs

- README
- API reference
- ADR
- changelog
- AI-friendly project summary
