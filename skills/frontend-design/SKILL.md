# Frontend Design

## Overview

Design web interfaces with deliberate hierarchy, interaction clarity, and visual character instead of default AI-safe layouts.

## Rules

- Ask before assuming palette, style, or layout direction when the request is open-ended.
- Avoid generic AI-safe defaults unless the existing system requires them.
- Let UX psychology and content hierarchy drive visual decisions.
- Match the design language to audience, brand, and product context.

## When to use

- component and layout design
- color and typography decisions
- UI polish and visual direction
- front-end interface critique

## Workflow

1. Clarify constraints: audience, brand, platform, content, and technical stack.
2. Choose visual direction intentionally rather than defaulting to clichés.
3. Define hierarchy through layout, color, spacing, and type.
4. Design states and interactions, not just static screens.
5. Verify readability, trust, and usability before polishing effects.

## Focus points

- user psychology
- hierarchy
- spacing and scale
- color strategy
- typography
- motion restraint
